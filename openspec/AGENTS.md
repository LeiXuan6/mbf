# OpenSpec Local Instructions

- Treat `openspec/specs/` as current implemented truth.
- Treat `openspec/changes/` as proposed or in-progress work.
- Do not edit current specs for a feature directly; write delta specs under a change first.
- Use requirement blocks with `SHALL` or `MUST`.
- Use scenario blocks with `GIVEN`, `WHEN`, and `THEN`.
- Keep change IDs kebab-case and action-oriented, for example `add-skill-schema-validation`.
